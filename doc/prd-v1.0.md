📄 产品需求文档 (PRD)：PureWorker--教师专属AI桌面工作台


1. 产品概述
产品名称：PureWorker

产品定位： 一款专为中小学教师及教研人员设计的、运行在电脑桌面端（支持 Windows / macOS）的智能化专属工作台。

核心理念： 告别传统的“对话框聊天”，本产品以**“自动化处理本地文件”和“沉淀长效师生记忆”**为核心。通过极简的拖拽交互，将教师从海量的文档排版、作业统计、表格填报和家校沟通中彻底解放出来。

目标用户： K12 各学科教师、班主任、教研组长及培训机构讲师。

2. 核心系统特性需求
记忆引擎（核心护城河）： 系统不再是“阅后即焚”的单次对话工具，而是具备长效记忆的数字大脑。

结构化档案记忆： 永久记住学生的姓名、班级、历次考试分数变动。

非结构化情境记忆： 记住老师日常录入的评语片段、学生高频错题图片、家校沟通的历史上下文。在后续下达指令时，系统会自动提取这些记忆作为参考依据。

无缝的文件吞吐流： 原生支持读取和输出教师常用的办公格式（Word、Excel、PPT、PDF、各类图像及音视频），实现“输入原始粗糙素材，一键输出合规标准文档”。

3. 基础教务管理模块（数据基座）
为 AI 提供精准的上下文支撑，系统需具备一套轻量化、可视化的教务管理看板。

3.1 班级与学科管理：

支持创建多个平行班级或不同年级分组。

支持绑定教师的任教科目，确保 AI 在处理内容时自带学科语境。

3.2 学生 360° 数字档案：

极速建档： 支持通过标准 Excel 模板一键导入全班名单及基础信息。

动态标签池： 允许教师快速为学生打上自定义行为/性格标签（如“理科强”、“需关注情绪”、“单亲家庭”）。

成长轨迹看板： 每个学生拥有独立主页，聚合展示其成绩波动曲线、历史典型错题本、以及教师的日常观察随记。

3.3 日程与课表管理：

内置轻量级工作日历，记录课程进度与教研会议。

支持将生成的教案、课件直接关联到特定日期的课次上，方便跨学期追溯复用。

4. 核心业务工作流模块（高频场景需求）
4.1 智能备课与教研中心
多源素材一键转 PPT：

需求： 教师圈选本地多个参考文档（PDF、Word 或网页截图），下达授课目标指令。

输出： 系统自动阅读所有素材，提炼知识大纲，生成一份逻辑清晰、带基础排版的演示文稿。

分层教学练习生成器：

需求： 导入一份标准版课后练习文档。

输出： 系统自动将其改写为两份衍生版本：包含详细步骤提示的“基础版”（针对后进生），以及包含高难度拓展题的“挑战版”（针对尖子生）。

音视频教研“榨汁机”：

需求： 拖入长达一小时的教研会议录音或公开课观摩视频。

输出： 自动识别语音，提取核心教学观点和重点发言，输出结构化的 Word 版会议纪要或评课报告。

4.2 作业、考评与题库管理
非标准作业批量结构化（碎活终结者）：

需求： 教师拖入几十张格式各异、光线不一的学生作业照片或扫描件。

输出： 系统自动提取照片中的作答内容并初步核对正误，最终输出一份带学号、姓名和得分的结构化 Excel 汇总表。

针对性错题重组与专属练习：

需求： 选择特定学生，输入指令（如：“提取李四近一个月的立体几何错题，并出5道相似题”）。

输出： 调取该生历史档案，变动错题的数字或物理情境，生成一份全新的专属 Word 练习卷及独立答案页。

4.3 班务与家校沟通

高情商家长沟通文案（带记忆注入）：

需求： 输入极简指令（如：“跟张三妈妈反馈他最近逃避交作业”）。

输出： 系统自动读取张三的档案记忆（如：近期有进步、家长较焦虑），生成一段先肯定优点、再委婉指出问题，并附带家校配合建议的得体微信话术。

期末评语批量生成：

需求： 结合期末成绩单和日常表现记录表。

输出： 批量生成全班几十份有理有据、不重样、包含具体细节的专属期末评语。

班会与校园活动全案策划：

需求： 输入活动主题（如“开课通知”）。

输出： 面向全员家长/学生的文案

5. 系统设置与个性化配置模块（中枢控制台）
为满足不同学校合规要求及教师个人习惯，提供深度的系统配置能力。

5.1 AI 供应商与算力配置：


自定义云端密钥： 支持配置第三方主流大模型厂商的服务密钥。


5.2 数据安全与隐私合规：

本地存储路径管理： 允许自定义核心数据库与记忆档案的本地存放盘符或加密文件夹。

数据生命周期管理： 提供一键打包导出/归档特定毕业届别/班级数据的功能，并支持对工作台残留数据的安全擦除。

5.3 教学上下文与身份预设：

教师身份名片： 预设教师的学段、学科及常用教材版本，避免后续下达指令时反复补充背景信息。

专属沟通语气库： 设定默认的沟通基调（如“严慈相济”、“鼓励为主”），或支持导入教师过往的评语范文，让 AI 深度模仿本人的行文风格。

校本评价体系配置： 自定义学校的考评标准（百分制、等级制等），统一 AI 处理成绩表格时的输出口径。

5.4 文档处理与校本模板管理：

专属红头文件库： 允许教师上传学校统一下发的官方空白模板（带有校徽的教案表、特定格式的试卷头）。AI 组卷或备课时，强制优先套用该校本模板。

默认输出偏好： 设定导出文件的默认格式（如统一默认导出便于修改的 .docx 而非 .pdf）及默认保存路径。

5.5 基础全局设置：

全局唤醒快捷键： 支持自定义快捷键，在任何软件界面均可一键呼出悬浮指令台。

后台常驻监控： 允许系统在后台静默运行，自动监控并处理特定“接收文件夹”内的新增作业图片。

5.6 智能体能力扩展与插件生态 (Skills & MCP Integrations)
本工作台采用“大脑（大模型）+ 手脚（Skills）+ 感官（MCP）”的解耦架构设计。

原子化动作引擎 (Built-in Skills)： 兼容Claude Skills。系统原生内置一套桌面级动作技能（包含但不限于：Office 文档原生读写、高精度数学计算环境、基础图像裁剪处理）。大模型可自主调度这些技能，实现从“输出文本”到“直接输出成品文件”的闭环。

第三方工具库 (Skill Store)： 支持一键安装垂直场景扩展技能（如：化学方程式渲染工具、英语口语发音评测工具、自动排课算法工具），弥补通用大模型在特定学科领域的短板。支持从文件/仓库导入

校内数据管道 (MCP Server)： 基于标准模型上下文协议，支持接入外部MCP。

5.7 快捷指令配置

支持配置快捷指令 可以在对话框以'/'的形式呼出


6. 交互与体验设计 (UX/UI)

6.1 设计原则

以"不想学新软件"的教师群体为核心设计对象（30-55岁，日常使用微信、Word、Excel），所有设计决策遵循以下原则：

30秒出价值： 打开应用即可拖入文件开始工作，无需前置配置或学习成本。

认得出优于想得起： 所有功能按钮必须同时包含图标与中文文字标签，禁止仅用图标表达功能含义。

结果导向优于对话导向： 用户的核心诉求是获得成品文件，而非维护一段对话记录。界面应以文件处理结果为主体，对话为辅助。

教师确认优于AI自动： AI 的所有输出均以"草稿"形态呈现，必须经教师明确确认后方可生效或导出，杜绝自动发送等高风险操作。

微信心智模型： 借用教师已熟悉的微信交互语言（可见按钮优先、三点菜单替代右键、红点通知等），降低认知门槛。

6.2 整体布局架构

采用可折叠的三栏式布局：

左侧导航栏（220px，可折叠为64px图标模式）：
功能模块入口（工作台、学生档案、快捷指令等），每个入口均为"图标+文字"形式。
最近处理的文件列表，方便快速回溯。

主工作区（自适应宽度，占据剩余空间）：
默认状态为"拖拽投放区"，包含醒目的文件投放提示、支持格式说明。
投放区下方常驻高频功能按钮组（如"批改作业""写评语""填报表格""备课出PPT""家长沟通"），解决用户"不知道该做什么"的空白焦虑。
文件投放后自动切换为任务内容视图（文件预览、处理进度、结果展示）。

右侧AI助手面板（360px，可折叠隐藏）：
实时展示AI执行轨迹与当前进度（人话描述，如"正在分析第5份卷子"）。
支持随时输入补充指令，微调AI行为。
底部提供随时中止按钮。

底部状态栏：
持续显示当前连接的AI模型、本地存储用量、脱敏保护开关状态，给予用户系统掌控感。

6.3 核心交互流

6.3.1 文件投放与智能识别

投放区同时支持三种输入方式：拖拽文件、点击选择文件、直接输入自然语言指令，不强迫用户选择单一路径。

文件拖入时，投放区边框产生发光脉冲效果，确认用户操作已被识别。

系统自动分析文件属性（类型、数量、内容特征），弹出对应的智能工作流推荐卡片（如"检测到30张作业照片，建议开始批改作业"），用户一键确认即可启动。

6.3.2 白盒化执行反馈

拒绝黑盒转圈。在执行长耗时任务时，右侧AI面板必须实时打印人类可读的执行轨迹：

使用"已完成/进行中/等待中"三态标记每个步骤。

每个步骤用自然语言描述（如"已按学号自动分组28组""正在识别第15/28组的作答内容"），禁止出现技术术语或英文错误码。

遇到需要人工介入的情况（如照片模糊、数据不足）时，主动暂停并在进度流中插入可点击的确认卡片，引导教师处理后继续。

显示预估剩余时间，提供随时中止按钮。

6.3.3 结果预览与确认

AI处理完成后，进入结果预览视图：

对于文档类输出（评语、沟通话术等），采用逐条卡片式呈现，每条独立提供"编辑""采纳""重新生成"三个操作按钮。

每条AI输出附带"依据标注"，标明该输出参考了哪些数据来源（如"依据：成绩记录4次、课堂观察3条"），建立教师对AI输出的信任感。

当AI置信度不足时（如学生档案数据过少），以醒目的警示标记提示"数据不足，建议补充后重新生成"，而非硬编低质量内容。

最终通过"全部采纳"或"导出"按钮完成输出，导出前支持选择格式（Word/Excel/PDF）及保存路径。

6.3.4 快捷指令系统

底部指令输入框支持以"/"前缀呼出快捷指令菜单，指令列表以"图标+名称+一句话说明"呈现（如"/评语 批量生成期末评语"）。

同时支持直接输入自然语言指令（如"帮我跟张三妈妈说他最近作业完成得不错但上课注意力不集中"），系统自动匹配对应工作流。

6.4 关键页面规格

6.4.1 工作台首页

页面主体为文件投放区，支持拖拽、点击、自然语言三种输入。

投放区下方展示高频功能快捷按钮组。

右侧AI面板默认展开，展示欢迎语与使用引导。

6.4.2 学生档案列表页

支持通过Excel模板一键导入全班名单。

列表支持按班级/学科筛选，支持搜索。

每行学生条目展示姓名、学号、自定义标签，点击进入个人档案。

6.4.3 学生个人档案页

顶部展示学生基本信息与自定义标签池（彩色药丸样式，支持点击添加）。

主体区域通过Tab页切换展示：成绩趋势（折线图）、教师观察记录（时间线）、家校沟通历史。

底部固定AI操作按钮（如"基于档案生成评语""分析成绩变动"），档案数据自动作为AI上下文注入。

6.4.4 AI结果预览页

逐条卡片式布局，每条AI输出为一个独立可操作卡片。

卡片内包含：生成内容正文、数据来源标注、编辑/采纳/重新生成按钮。

页面顶部提供"全部采纳"与"导出"批量操作。

6.4.5 设置页

分区卡片式布局，每个设置区块为独立卡片：AI模型配置、数据隐私与脱敏、教师身份预设、文档模板管理、全局偏好。

技术参数（如Temperature、Top-P）对用户隐藏，仅暴露语义化选项（如"创意模式/严谨模式"切换）。

6.5 视觉规范

6.5.1 配色体系

主色（品牌蓝 #2B6CB0）：传达专业与信赖感，用于主要按钮、导航高亮、品牌标识。

辅助色（暖橙 #E8853D）：传达温暖与教育关怀，用于重要提示、功能引导。

功能色：成功绿 #38A169（完成/正确）、警告黄 #D69E2E（注意/待确认）、错误红 #E53E3E（异常/失败）。

背景色采用纸张白（#FAFAF8）而非纯白（#FFFFFF），减少长时间使用的眼疲劳，贴近实体纸张的视觉感受。

6.5.2 字体规范

使用系统字体栈，不捆绑额外字体文件：macOS使用PingFang SC，Windows使用微软雅黑。

字号规格：
页面标题：22px / SemiBold
区块标题：16px / Medium
正文内容：14-15px / Regular（面向30-55岁用户，绝不低于14px）
辅助说明：13px / Regular（仅用于次要信息，颜色降灰）
数据表格：13-14px / Regular（行高不低于40px，保证可点击性）
AI生成文本：15px / Regular（行高1.75，保障阅读舒适度）

6.5.3 圆角与间距

卡片圆角：12px（传达亲和力，避免方正的冰冷感）。
按钮圆角：8px。
内间距最小值：16px（保证触控友好性）。
卡片间距：16-24px。

6.5.4 图标系统

基础图标库采用Lucide Icons（线条统一、风格清晰）。

需定制教育场景专属图标：学生、作业、成绩、家长、班级、表格等。

所有图标必须配合中文文字标签使用，禁止单独图标表达功能。

活跃状态使用填充（Filled）图标，非活跃状态使用线框（Outline）图标，提供清晰的视觉反馈。

6.6 动效规范

页面切换：水平滑入 + 淡入，时长200ms。

拖拽文件悬停：投放区边框发光脉冲 + 轻微放大，持续至松手。

AI处理中：骨架屏闪烁（Shimmer）效果，配合逐行淡入的进度文字（300ms/行）。

任务完成：绿色对勾 + 轻微弹跳，时长400ms。

文件导出成功：底部Toast非侵入式滑入通知，3秒后自动消失。

禁止事项：旋转加载圈超过3秒不带文字说明；弹窗打断当前操作流；任何跳跃或抖动类动效。

6.7 设计禁区

禁止使用纯图标导航，所有功能入口必须图标+文字。

禁止将核心功能放置在右键菜单中，所有操作必须有可见按钮入口。

禁止向用户暴露技术参数（Temperature、Top-P、Token数等），一律转化为语义化选项。

禁止AI输出自动生效，所有输出必须经"教师确认"环节。

禁止使用英文错误提示，所有系统反馈必须为中文自然语言（如"网络超时，请稍后重试"而非"Error: timeout"）。

禁止正文字号小于13px，AI生成内容字号不低于15px。

禁止使用侵入式弹窗打断操作流，统一使用底部Toast通知。
